Getting Started: MetMSline Instructions

Purpose

MetMSLine and associated software functions were created in the R language to attempt to achieve improved standard operating procedures and good laboratory practice for efficient and objective untargeted metabolomics analysis of large-scale (>300 samples) high-resolution LC-MS datasets.

You will need to install the latest version of R (available here: R.download.page) and also following R installation RStudio is recommended for its user-friendly interface (available here: Rstudio)

Required packages

-Initially install all R package dependencies by typing or pasting these lines of code into the command line of the R or RStudio console, you will only need to do this once per computer operating system:

install.packages(c("devtools","httr","zoo", "gplots", "ggplot2", "pls", "sfsmisc", "RColorBrewer", "readMzXmlData"))
source("http://bioconductor.org/biocLite.R")
biocLite("pcaMethods")

What does it do?

-As long as all packages installed successfully: simply paste or type this line of code into your R/ RStudio command line and you can see for yourself:

eval(parse(file="http://bit.ly/18nUgrl"));Auto.Install.Synth.test()

The above command will install all the functions of MetMSLine: PreProc.QC.LSC.R, Auto.PCA.R, Auto.MV.Regress, Auto.MS.MS.match.R and DBAnnotate.R (folder name = MetMSLine.functions) in whichever directory in R/RStudio you are currently working in. To find out what your current directory in your R session is simply paste or type in the command line:

getwd()

TROUBLE SHOOTING TIP

If MetMSLine does not install in windows right click the R/RStudio icon on the desktop->properties and add --internet2 in the Target box after .exe" (i.e. .exe" --internet2) and then click ok. This should resolve the problem.

The MetMSLine installation also includes automatic processing of some synthetic high resolution LC-MS data (5000 MS features, 80 samples, 20 Quality Controls) and synthetic continuous Y-variable data (10 Y-variables) with default settings.

NB. You can monitor progress by watching the new directories of results as the data is processed and the saving of all MetMSLine functions as it installs

The function Auto.MS.MS.match.R is the only function not illustrated by the synthetic dataset as the data is simulated it is not possible to match random features to real MS/MS fragmentation data.
